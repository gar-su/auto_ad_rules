# 需求文档：推广链接批量搜索

## 版本

| 版本 | 日期 | 作者 | 变更 |
|------|------|------|------|
| v1.0 | 2026-05-27 | - | 初稿 |

## 背景

控损规则创建页面中，推广链接模块支持按链接 ID 搜索。现有搜索栏仅支持单个 ID 输入。运营人员在实际使用中常需一次查询多个链接 ID（如从 Excel 复制一列 ID），逐个搜索效率极低，需增加批量搜索入口。

## 目标

在推广链接搜索栏增加批量搜索模式，支持用户一次性输入多个链接 ID 进行搜索，同时保留原有单 ID 搜索能力，两种模式可自由切换。

## 范围

- **涉及页面**：控损规则创建 — 控损对象 — 推广链接
- **涉及文件**：`index.html`
- **不涉及**：账户范围、短剧范围的搜索栏（本次仅推广链接）

---

## 功能详述

### F1. 模式切换

| 属性 | 描述 |
|------|------|
| 触发 | 点击搜索栏右侧"批量搜索"文字链接 |
| 前置 | 当前为普通搜索模式（默认） |
| 后置 | 进入批量搜索模式 |
| 说明 | 搜索栏区域的切换入口始终可见，不因模式变化而隐藏 |

- 普通模式：`[单行输入框] [搜索按钮] [批量搜索]`
- 批量模式：`[多行文本框] [搜索按钮] [普通搜索]`

切换入口文字随当前模式变化：
- 普通模式 → 显示"批量搜索"
- 批量模式 → 显示"普通搜索"

### F2. 批量搜索界面

进入批量模式后：
1. 原有单行搜索栏隐藏
2. 展开一个多行文本输入框（textarea），默认 3 行高度，可手动拖拽调整
3. 文本输入框下方放置搜索按钮
4. placeholder 提示："每行一个，支持输入链接ID批量搜索"

### F3. 搜索执行

- 点击搜索按钮后，将 textarea 内容按换行、逗号、空格三种分隔符拆分为 ID 列表
- 空字符串和纯空白行自动过滤
- **当前阶段**：搜索逻辑暂不实现（原型阶段），仅做模式切换交互
- **后续实现**：将 ID 列表传入表格过滤逻辑，高亮匹配行

### F4. 退回普通模式

- 点击"普通搜索"链接切换回普通模式
- textarea 隐藏，原有单行搜索栏恢复显示
- **数据带回**：textarea 第一行内容（按分隔符拆分后的第一个 ID）自动填入单行输入框
- 其余行丢弃（用户已确认接受此行为）

## 状态说明

| 状态 | 搜索栏 (.search-bar) | 批量区 (.batch-area) | 切换入口文字 |
|------|---------------------|---------------------|-------------|
| 普通模式（默认） | display:flex | display:none | 批量搜索 |
| 批量模式 | display:none | display:flex | 普通搜索 |

## 验收标准

- [ ] **AC1**：点击"批量搜索"，单行输入框隐藏，多行文本框展开
- [ ] **AC2**：切换入口文字同步变为"普通搜索"
- [ ] **AC3**：点击"普通搜索"，多行文本框隐藏，单行输入框恢复
- [ ] **AC4**：退回普通模式时，textarea 第一行内容自动填入单行输入框
- [ ] **AC5**：textarea placeholder 显示"每行一个，支持输入链接ID批量搜索"
- [ ] **AC6**：textarea 默认 3 行高度，支持垂直拖拽调整
- [ ] **AC7**：切换入口始终可见，不因模式切换而消失

## 技术实现要点

| 项目 | 说明 |
|------|------|
| 切换实现 | JS 事件监听 `.batch-toggle`，切换 `.search-bar` 和 `.batch-area` 的 `display` 属性 |
| DOM 位置 | `.batch-toggle` 位于 `.search-bar` 和 `.batch-area` 同级（避免随 searchBar 一同隐藏） |
| 数据带回 | `textarea.value.split(/[\n,]+/)[0]?.trim()` |
| 样式 | `.batch-area` 使用 `display:flex; align-items:flex-start; gap:10px` |
| 文本域样式 | `.batch-textarea` max-width:600px, resize:vertical |
