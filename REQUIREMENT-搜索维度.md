# 需求文档：短剧范围搜索维度

## 版本

| 版本 | 日期 | 作者 | 变更 |
|------|------|------|------|
| v1.0 | 2026-05-27 | - | 初稿 |
| v1.1 | 2026-05-27 | - | 右侧已选表格保留短剧名称列，转写逻辑改为全量映射 |
| v1.2 | 2026-06-02 | - | 新增批量搜索模式 |

## 背景

短剧范围模块的搜索栏仅支持按短剧名称搜索。实际业务中，运营人员常需通过剧本编号定位短剧（剧本编号来自内部剧本管理系统，如 `SC20240501001`），现有单维度搜索无法满足需求。

## 目标

在短剧范围搜索栏增加搜索维度选择，支持按短剧名称、剧本编号、剧本名称三种维度切换搜索，并相应扩展表格列展示剧本编号。

## 范围

- **涉及页面**：控损规则创建 — 控损对象 — 短剧范围
- **涉及文件**：`index.html`
- **不涉及**：账户范围、推广链接的搜索栏

---

## 功能详述

### F1. 搜索维度选择器

| 属性 | 描述 |
|------|------|
| 位置 | 搜索栏最左侧，搜索输入框之前 |
| 形态 | `<select>` 下拉框 |
| 选项 | 短剧名称 / 剧本编号 / 剧本名称 |
| 默认值 | 短剧名称 |
| 样式 | 与搜索输入框视觉连接（共用 border，无间隙） |

状态示意：

```
[短剧名称 ▾][________________________][搜索]
[剧本编号 ▾][________________________][搜索]
[剧本名称 ▾][________________________][搜索]
```

### F2. 输入框占位提示联动

| 维度 | placeholder |
|------|-------------|
| 短剧名称 | 请输入短剧名称 |
| 剧本编号 | 请输入剧本编号 |
| 剧本名称 | 请输入剧本名称 |

切换维度时，输入框 placeholder 即时更新。

### F3. 剧本编号列

| 属性 | 描述 |
|------|------|
| 位置 | 左侧候选表格，checkbox 与剧本名称之间 |
| 列头 | 剧本编号 |
| 数据 | 剧本编号字符串，如 `SC20240501001` |
| 联动 | 右侧已选表格同步展示剧本编号列 |

左侧候选表格列变更：

```
变更前:  ☐ | 剧本名称 | 短剧名称 | 语种
变更后:  ☐ | 剧本编号 | 剧本名称 | 短剧名称 | 语种
```

右侧已选表格列变更：

```
变更前:  剧本名称 | 语种 | 操作
变更后:  剧本编号 | 剧本名称 | 短剧名称 | 语种 | 操作
```

### F4. 转写逻辑适配

左表数据列与右表数据列一一对应（均为 4 列：剧本编号、剧本名称、短剧名称、语种），转写时无需跳列。

数据映射：

```
左表 col(1) 剧本编号 → 右表 col(1) 剧本编号
左表 col(2) 剧本名称 → 右表 col(2) 剧本名称
左表 col(3) 短剧名称 → 右表 col(3) 短剧名称
左表 col(4) 语种     → 右表 col(4) 语种
```

---

### F5. 批量搜索

| 属性 | 描述 |
|------|------|
| 触发 | 点击搜索栏右侧"批量搜索"文字链接 |
| 前置 | 当前为普通搜索模式（默认） |
| 后置 | 进入批量搜索模式，单行输入框隐藏，多行文本框展开 |
| 说明 | 维度选择器在两种模式下始终可见，切换入口不随模式变化而消失 |

- 普通模式：`[维度▾][单行输入框........][搜索] [批量搜索]`
- 批量模式：`[维度▾][多行文本框........][搜索] [普通搜索]`

切换入口文字随当前模式变化：
- 普通模式 → 显示"批量搜索"
- 批量模式 → 显示"普通搜索"

**批量搜索界面：**
1. 维度选择器保持可见（告知用户当前按哪个维度搜索）
2. 单行输入框隐藏，展开多行文本框（textarea），默认 3 行高度，可手动拖拽调整
3. placeholder 随维度联动：
   - 短剧名称 → "每行一个，支持输入短剧名称批量搜索"
   - 剧本编号 → "每行一个，支持输入剧本编号批量搜索"
   - 剧本名称 → "每行一个，支持输入剧本名称批量搜索"

**退回普通模式：**
- 点击"普通搜索"链接切换回普通模式
- textarea 第一行内容（按换行/逗号/空格拆分后的第一个值）自动填入单行输入框
- 其余行丢弃

**搜索执行（当前阶段）：**
- 搜索逻辑暂不实现（原型阶段），仅做模式切换交互

## 交互清单

| ID | 触发 | 行为 |
|----|------|------|
| I-DIM-1 | 下拉切换维度 | 搜索输入框 placeholder 即时更新 |
| I-DIM-2 | checkbox 勾选行 | 新列结构下的数据全量转入右侧（剧本编号+剧本名称+短剧名称+语种） |
| I-DIM-3 | 点击"批量搜索" | 单行输入框隐藏，多行文本框展开；维度选择器保持可见；切换入口文字变为"普通搜索" |
| I-DIM-4 | 点击"普通搜索" | 多行文本框隐藏，单行输入框恢复；textarea 第一行内容自动填入输入框；切换入口文字变为"批量搜索" |

## 技术实现要点

| 项目 | 说明 |
|------|------|
| 维度选择器 CSS | `.search-dim`：border-radius:4px 0 0 4px, border-right:none, min-width:100px |
| 输入框衔接 CSS | `.search-dim + .search-input`：border-radius:0, border-left:none |
| placeholder 切换 JS | 监听 `.search-dim` change 事件，根据 value 映射 placeholder 文案 |
| 转写逻辑 | 左右表数据列一一对应（均 4 列），随 rightCols 自动适配 |
| 表格列新增 | thead 和 tbody 中每个 tr 追加一个 `<td>` |
| 批量搜索布局 | `drama-search-row` flex 容器包裹维度 select + search-bar + batch-area，维度 select 始终可见 |
| 批量切换 JS | 监听 `#dramaBatchToggle`，切换 `.drama-search-bar` / `.drama-batch-area` 显隐 |
| textarea placeholder 联动 | 维度 change 事件同步更新 `.batch-textarea` placeholder，含三组映射 |
| 数据带回 | `textarea.value.split(/[\\n,]+/)[0]?.trim()` |
